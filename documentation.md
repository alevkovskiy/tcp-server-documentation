"""
Документация: кэш-сервис

Этот сервер поддерживает простой текстовый протокол
и реализует базовые команды кэширования с поддержкой TTL.

Формат ответов:
- Успех: "OK", "значение"
- Отсутствие значения: "nil"
- Ошибка: "ERROR. <сообщение>"

Все команды завершаются символом новой строки при отправке клиентом.
Сервер автоматически закрывает соединение после обработки одной команды.
"""

    "SET"
    SET key value [EX seconds] - Сохраняет значение(value) по ключу(key). Опционально задаётся время жизни (TTL) в секундах.
    
    Примеры:

      SET name Alice - устанавливает полю name значение Alice
      SET greeting "Hello world" EX 30 - устанавливает полю greeting значение "Hello world" c TTL = 30

    Ответ: "OK"

    "GET"
    GET key - Возвращает значение по ключу или "nil", если:
                                                            - ключ не существует
                                                            - TTL истёк

    Пример:

      GET name

        Ответ: "Alice" или "(nil)"
        """,


    "EXISTS"
    "EXISTS" key - возвращает 1(Истина) если заданному ключу(key) поставлено в соответствие какое-либо значение и 0(Ложь) если такого значения нет
