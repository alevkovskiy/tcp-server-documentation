# Архитектура кэш-сервера

## Общая структура
Сервер реализован на чистом Python с использованием стандартной библиотеки (`asyncio`).  
Архитектура следует принципу **разделения ответственности**:

| Компонент | Ответственность |
| `server.py` | Запуск TCP-сервера, приём подключений, маршрутизация команд |
| `commands.py` | Логика обработки команд (`GET`, `SET`) |
| `storage.py` | In-memory хранилище данных с поддержкой TTL |
| `converter.py` | Парсинг входных строк в структуру `(команда, аргументы)` |

Сервер работает в **однопоточном асинхронном режиме**: каждое подключение обрабатывается в отдельной корутине, что обеспечивает эффективную обработку множества клиентов без блокировок.

Вся информация о командах, их использовании и ответах сервера находится в файле documentation.md

## Хранилище и TTL

- Данные хранятся в памяти в структуре:
  ```python
  {key: {"value": str, "expiry": float | None}}

- Решение поддерживает "ленивую" очистку по TTL (при вызове GET проверяется TTL и при необходимости запись удаляется)

## Масштабируемость

Все команды являются объектами callable и хранятся в словаре парами {"Команда" : функция}, в связи с чем для добавления новых функций достаточно создать соответствующий объект callable и добавить пару ключ - значение в словарь. Такое решение обеспечивает лучшую масштабируемость
